xampp与WordPress的安装

2014-08-06

既然要搞wp，首先考虑先在本地安装好，弄好之后再上传到web上去，所以先安装运行环境——xampp。

大名鼎鼎的php集成软件包xampp

不用介绍了，官方网站+下载地址：

https://www.apachefriends.org/zh_cn/index.html

下载之后直接运行安装，于是出现了以下杯具的一幕：

php自5.5版本起不再支持Windows XP和2003系统o(╯□╰)o

为XP娘默哀一下。。

果断重新去到官网下载了5.4版本，可以顺利安装，不过之前出现了如下提示：

辛苦的翻译君/(ㄒoㄒ)/~~ ：看起来您有一个杀毒软件在运行，这有可能减慢或者干预xampp的运行，请访问以下链接了解更多

继续安装？

无视之(☆-ｖ-)，点“YES”

next

直接默认了，即便用不到，也没啥影响不是

继续next

安装过程中程序有可能会去下载必须的运行环境vc2008，并且坑爹的是直接下载存放到了

安装目录的上级目录，这点需要注意。

安装完成之后可以手动删除。

启动可能出现端口占用的情况，检查一下当前端口情况，如果不知道什么软件占用，就重启下看看

xampp默认的localhost根目录是xampp/htdocs，

因此我们可以直接把wordpress的文件解压至该目录下

如果图方便可以像我一样，修改htdocs下的index.php文件，将其中的xampp改为wordpress（这里的wordpress是指解压到这里的wordpress文件夹名称，可变化的）或者修改httpd.conf中的DocumentRoot路径，就可以直接在浏览器中输入localhost跳转到wordpress了。否则就只能输入localhost/wordpress来访问，缺点是点击apache的admin也会跳转到wordpress，斟酌使用。

输入localhost或者localhost/wordpress开始安装wordpress：

点击“创建配置文件”按钮

这时可以先把这个页面放一放，打开xampp control panel，点击admin，进入phpMyAdmin。

phpMyAdmin是一个web端管理mysql数据库的工具，xampp是集成了的，如果是个人服务器并且没有的话，可以额外下载一个，解压到apache服务器的web应用目录下安装使用。

页面会有如下提示，如果是部署到服务器还是建议更改一下密码，不过我这里是本机，就完全可以无视了。

点击New，创建一个新表用来存储WordPress数据，名称任意不过建议WordPress相关，让自己清楚是那张表。

然后把相关的数据库信息填写到之前的WordPress安装页面中，安装前基本的配置信息也就完成了，果断点击进行安装。

安装后填写一些博客标题、管理员账号等基本信息，就可以正常使用WordPress了。

wordpress phpmyadmin xampp
